SPINE manual

 

SPINE (Signal Processing In Node Environment) is a framework that allows dynamic configuration of feature extraction capabilities of WSN nodes via an OtA protocol.      

The aims of the framework are:

  • Providing an open source project that may integrate different hardware platforms;
  • Provide new TinyOS libraries and utilities for easy application development;
  • Support a OtA protocol for managing and monitoring WSNs.

 

The framework is built by different parts.

The main ones are the node-side one ,which has been developed in TinyOS2.0, and the server side one, which has been implemented in a Java environment for PC which is supposed to communicate to the WSN thought a classical TinyOS2.0 base station.

 

Overview

 

As previously said, SPINE has a tinyos on node side and a java server side: the code for make those two parts working has been placed into different subfolders.

In the following, the main characteristics of both are briefly presented.

 

Spine_nodes

 

The spine_nodes folder contains source code written for the TinyOS environment.

The folder contains as well a README file to follow for the installation of this code on TinyOS2.x folder you must already have on the computer (see http://www.tinyos.net/tinyos-2.x/doc/html/install-tinyos.html  for details).

 

The following modules have been developed in TinyOS 2.0 http://www.tinyos.net/ to enable the on mote signal processing:

 

Circolar Buffer

Since no generic structure for storing data was present in TinyOS, a circular buffer has been implemented with the aim of making “store” and “loan” operations transparent to the user.

 

The code may be found in tos\system\utils.

In particular:

“GenericCircularBuffer”  is the component that provides an implementation of a generic circular buffer. It can handle 8, 16, 32 bit values, simply instancing it in the proper manner at compile time. It's also possible to define over the buffer a window frame and a window shift.

“GenericSharedCircularBuffer”, with respect to the “GenericCircularBuffer” component, it requires as parameter for its initialization the array in which the elements will be stored. This allows sharing the array among several GenericSharedCircularBuffer.

 

 

Utility  Libraries

Some mathematical functions (such as sorting an array and square root calculation) are defined to allow on mote signal processing using those functions.

 

The code may be found at in tos\system\utils.In particular:

“SortC” component provides several kind of array sorting algorithms;

 “MathC” provides different methods for providing the truncated integer square root of a given parameter.

 

Feature Extraction Library

Since the first main goal of the project into which the framework has been developed was to do on node feature extraction of sensors’ data, some libraries for feature extractions were needed. Therefore the Feature Selection Library has been developed in a modular fashion, dividing the code between linear and non linear feature calculation.

 

The code may be found at tos\system\signalProcessing\featExtractors where 2 folders contain linear and non linear feature calculation.

 

The linear features provided are:

Amplitude, which gives the biggest variation of the signal and it is calculated as the maximum value of the signal minus the mean value of the signal itself

Central Value, which is the value of the signal which is in the middle of the vector

Mean, the mean value over the entire vector

Max, the maximum value over the entire vector

Min, the minimum value over the entire vector

Range, which is the difference between the maximum and the minimum value of the vector.

 

The non linear features provided are:

Median

Variance

Standard deviation

Root mean square

 

Job Engine

A component called Job Engine has been developed and implemented in TinyOS. This component acts as a dispatcher and defines the type of jobs the application need to execute and then it is able of executing them by calling the required component.

The code for the Job Engine component may be found in the application folder.

 

AMP (Activity Monitoring features selection Protocol)

The framework comes not only with the functionalities explained in the previous lines, but with a full defined protocol that defines the interaction the node may have with the base station node.

In details, it provides the following functionalities:

Service Discovery and Advertisement: whenever the base station needs to know which sensors are presents and which kind of service they can provide, it can broadcast the request message. On the nodes’ side, they will reply with their functionalities (i.e the sensors they supports, the features they can calculate…).

Feature Activation and Removal: the base station will command the nodes to send a proper set of features and other settings (sampling rate, window, shift)

Battery Info Request and Response: the base station will be able to check the level of the power of each node.

 

The code for building and parsing the packets defined by the protocol may be found at \tinyos-2.x\tos\system\amp.

To make this framework work with “UC Berkeley ,TelecomItalia ,Tele-Immersion Lab boards”  connected to Telosb motes.

 

Spine_serverApp

 

Spine_serverApp folder contains the source code for a java application at the central node is able of sending and receiving packets according to the AMP protocol.

The folder is organized as follow:

·        \src\demo contains the source code for a simple test application that may be used as template for a new project

·        \src contains the source code for the AMP(Activity Monitoring features selection Protocol) protocol interface

·        \lib contains 1 .jar file:

o       spine.jar which contains the binary version of the \src.

·        \doc the javadoc as a documentation of the spine_serverApp

 

Note that tinyos.jar MUST be included in the project as well. It can be found in the TinyOS2.0 tree (in the tinyos2.x\support\sdk\java folder or downloadable at http://tinyos.cvs.sourceforge.net/tinyos/tinyos-2.x/support/sdk/java/).

 

Users should be able to manage the nodes starting from the test application which gives a layer of abstraction of the underneath mechanisms.

 

Installing SPINE

 

Basically, the steps a user should follow to have SPINE working are:

·        Extract SPINE.tar. SPINE folder will be created containing this documentation, licensing information and 2 folders: Spine_nodes and Spine_serverApp.

Now, the installation of the framework is dividend into two step: the first one will regard telos motes tinyos programming and the second on PC java programming.

 

Nodes’ side

·        The user should have tinyos-2.x installed

·        Open Spine_nodes folder and follow instruction in the README file

 

1.      Copy 'Spine_nodes' directory into your 'tinyos-2.x-contrib' directory (should be at the same level that tinyos-2.x folder)

 

2.      Make sure you have 'BaseStation' directory into your 'apps' directory in tinyos-2.x.

 

3.      Connect a telosb mote to the PC and type "make telosb install,x" from "...\apps\FeatureSelectionAgent" path to program the terminal (sensor) mote         

Note: x is the node’s ID. Flash 2 different nodes with IDs “1” and “2”.

 

4.      Connect another mote to the PC (no sensor board needed) and type "make telosb install,y" from "...\apps\BaseStation" path to program the sink mote (base station) connected directly to the PC.

note: y must be replaced with a number; usually “0” is used for the base station

·        Now you should have 3 nodes programmed: 2 with the FeatureSelectionAgent and 1 with the BaseStation. Make sure that they all communicate on the same channel (channel 12 is used as default in the FeatureSelectionAgent app) and that the 2 nodes are equipped with the sensor board UC Berkeley/Tele-immersion Lab/Telecom Italia.

 

PC side

           

If you are using Eclipse SDK:

·        Create a Java project

·        Add to the project the source demo file SPINETest.java you find in SPINE\Spine_serverApp\src\demo

·        Add 2 external .jar in the Java Build Path:

o        SPINE\Spine_serverApp\lib\spine.jar

o       tinyos.jar (in the tinyos2.x\support\sdk\java folder or downloadable at http://tinyos.cvs.sourceforge.net/tinyos/tinyos-2.x/support/sdk/java/)

·        Note that the source code of the AMP protocol is provided as well in the \src folder. Instead of adding the spine.jar, the user may want to include the source files and compile them into its own project.

·        Before running the demo, open SPINETest.java and modify the setting according to your needs (PORT number).

·        Now you are ready for running SPINE!

 

Running SPINE

 

  • Connect the BaseStation to your PC
  • Power on the nodes with the sensor boards
  • Run SPINETest.java as a Java Application from Eclipse

 

Here you are!

The SPINETest.java application performs the following tasks:

  • Send a OtA request to discovery all the nodes in the network and their services
  • Nodes in the network will reply with their capabilities (e.g. which sensors they have on board and which features they are able to calculate)
  • Then, from the java app you can request different features to different nodes. In this example 5 features (mean X,Y,Z accelerometer, min and max X accelerometer) are asked to node 1 only one (min X accelerometer) to node 2.

 

You can now create your own application!
